# Horary App - Minimal MVP Plan

## Context

This horary astrology app is currently a working prototype with:
- Core chart calculations (7 classical planets + houses + aspects)
- Aspect motion analysis (applying/separating aspects)
- LLM-powered readings via local Ollama
- Vue 3 chat interface
- Reading history in localStorage

**The Goal:** Get to a shareable MVP quickly by fixing bugs, improving reliability, and making deployment easy.

**User Priorities:**
- Minimal scope - no auth, keep localStorage, focus on stability and deployment
- Multi-provider LLM support (Ollama, OpenAI-compatible APIs, Claude)
- Support open standards (OpenAI-compatible API) + best quality option (Claude)
- Give users choice based on their values, budget, and needs
- Save backend/auth for post-MVP

**Why Multi-Provider:**
- **Ethical alignment:** Don't force users to use OpenAI - support alternatives
- **User choice:** Privacy-focused users use Ollama, quality-focused use Claude, budget-conscious use Groq
- **Open standards:** OpenAI-compatible API is industry standard (one SDK, many providers)
- **Quality focus:** Claude excels at nuanced interpretation (perfect for horary readings)
- **Future-proof:** Easy to add more providers later

```typescript
export function calculateEssentialDignities(
  planet: Planet,
  absoluteDegrees: number,
  isDayChart?: boolean
): EssentialDignity {
  const { sign, degreeInSign } = getSignFromDegrees(absoluteDegrees);
  
  // Check rulership
  const isRuler = RULERSHIPS[planet].includes(sign);
  
  // Check exaltation (within 5° of exact degree)
  const exaltation = EXALTATIONS[planet];
  const isExalted = exaltation.sign === sign;
  
  // Calculate score
  let score = 0;
  if (isRuler) score += 5;
  if (isExactExaltation) score += 4;
  if (triplicityRuler) score += 3;
  if (isDetriment) score -= 4;
  if (isExactFall) score -= 5;
  
  return { isRuler, isExalted, score, strength, description };
}
```
